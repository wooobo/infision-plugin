# Kling 정합 (캐릭터 일관성·드리프트 방지)

같은 인물/객체를 여러 컷·여러 생성에 걸쳐 변하지 않게 유지하는 법. 어휘는 영문 그대로 프롬프트에 넣는다.

## 핵심 — 레퍼런스로 정체성을 고정

- **이미지 레퍼런스 최대 4장**(정면/측면/후면/디테일) → 360° 정합. 깨끗한 배경일수록 추출이 강하다.
- **영상 레퍼런스 3~8초** → 외형 + 모션 + 보이스 톤 동시 추출.
- 정합 레퍼런스는 r2v 의 참고 이미지(≤7) / i2v 의 첫·끝 프레임 슬롯에 들어간다 — 어떤 이미지를 어느 슬롯에 넣을지는 사용자가 캔버스에서 연결한다.
- 한 작업에 **요소 최대 3개** 바인딩.

## 프롬프트 레벨 정합

- **같은 캐릭터엔 동일 디스크립터 문자열을 그대로 반복.** `brown` ↔ `brunette` 처럼 동의어로 바꾸면 드리프트한다.
- 형식: `[Character description] + [action/pose] + [setting] + [style/quality]`. 캐릭터 묘사는 고정, action·setting만 바꾼다.
- 드리프트 네거티브: `morphing features, changing clothes, different eye color, inconsistent face`.
- **마스터 이미지 전략**: 고품질 포트레이트 1장을 먼저 확보하고, 이후 모든 생성에서 그 이미지를 레퍼런스로 재사용한다 — 같은 마스터를 여러 단계에 물리는 연결은 사용자가 캔버스에서 한다.

## 컷/클립을 넘는 정합

- **멀티샷(≤6컷/15초)** 안에서는 바인딩이 유지된다 — 와이드→클로즈업→오버숄더 동일 인물.
- **첫·끝 프레임(Start+End)** 을 주면 보간이 강제돼 흔들림이 준다.
- 클립을 이어 붙일 땐 **동일 화질 mode 로** — mode 가 바뀌면 광/질감/물리가 어긋난다.
- 5초를 넘어가면 "복사의 복사"로 정체성이 누적 변형된다. 길게 갈수록 레퍼런스 바인딩이 필수.

## 멀티 캐릭터 (3+)

- 인물이 많을수록 정합이 어렵다. **물리적 접촉(악수·물건 주고받기)** 장면이 가장 약하다 — 가능하면 인물을 공간적으로 떼어 배치.
- 인물별로 따로 레퍼런스 바인딩한다. 한 프롬프트에 여러 정체성을 욱여넣지 말 것.
- 단계 분해와 정합: 한 영상에 정체성 보존 + 외형 교체 + 큰 움직임을 **동시에** 요구하면 잘 안 나온다 → 단계로 쪼개는 게 맞다. 앞 단계에서 확정된 외형은 다음 단계 프롬프트에서 다시 묘사하지 말고, 이미 변환된 결과를 레퍼런스로 받는 전제로 쓴다(단계를 잇는 연결은 사용자가 캔버스에서 한다).
