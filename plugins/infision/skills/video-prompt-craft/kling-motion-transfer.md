# Kling 모션 트랜스퍼 / 모션 컨트롤

레퍼런스 영상의 움직임(스켈레톤·타이밍·물리)을 정지 캐릭터 이미지에 입힌다. 안무·제스처·특정 동작을 정확히 재현할 때.

## 입력 3종 (역할 분리)

| 입력 | 역할 | 넣을 것 |
| --- | --- | --- |
| 모션 레퍼런스 영상 | 움직임 청사진 | 실제 사람 퍼포먼스, 3~30초 |
| 캐릭터 이미지 | 시각 정체성 | 정지 이미지, 레퍼런스 영상과 프레이밍 일치 |
| 텍스트 프롬프트 | **연출만** | 장면·조명·스타일·분위기 — **동작은 쓰지 않는다** |

텍스트 슬롯 예(동작 묘사 금지):
```
cinematic [lighting], [environment], [visual style] aesthetic, volumetric fog, 4K resolution
```

## 오리엔테이션 모드

- **Character Orientation Matches Video** — 풀바디 안무/카메라 무브/복잡한 동작, 길게(~30초).
- **Character Orientation Matches Image** — 토킹헤드/포트레이트, 캐릭터가 원본 포즈에 고정(~10초).

## 규칙

- **프레이밍 일치가 가장 큰 실패 원인**: 풀바디 영상엔 풀바디 이미지, 클로즈업엔 클로즈업 이미지. 어긋나면 warping/clipping.
- 캐릭터 이미지에 **여백**을 둬서 큰 제스처가 잘리지 않게.
- 손동작이 있으면 캐릭터 이미지에서 **손이 보이게**(주머니/가림 → 손가락 환각).
- 모션 레퍼런스: 중간 속도·적은 이동·컷 없음·카메라 고정이 가장 깨끗하다. 빠른 회전 회피.
- 단일 인물 레퍼런스(멀티면 가장 큰 피사체를 추적).

## 트러블슈팅 (재촬영 전 스위치)

1. 오리엔테이션 모드 토글 — "이상하게 움직임"의 1순위 처방.
2. 프레이밍 재정렬(포트레이트↔풀바디 불일치 → warp/shake).
3. 손이 또렷한 캐릭터 이미지로 교체.
4. 더 느리고 깨끗한 레퍼런스 클립으로.

## 변형 스케일링

하나의 hero 모션 레퍼런스를 고정하고, 캐릭터 이미지나 텍스트(환경·조명·지역)만 바꿔 시즌/지역 변형을 양산한다 — 동작을 다시 안 찍어도 됨.

> 주의: 모션 컨트롤/트랜스퍼는 Kling 의 특정 모드 기능이다. 이 모드를 쓸 수 없는 상황이면 같은 의도를 i2v(첫·끝 프레임) + 모션 프롬프트로 근사한다 — `kling-camera-motion` 참고.
