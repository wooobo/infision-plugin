# Kling 트러블슈팅·품질 최적화

결과가 기대를 벗어날 때 증상→처방. 생성 전 품질 체크도 포함.

## 증상 → 처방

| 증상 | 처방 |
| --- | --- |
| 막연/일반적 결과 | subject+action+environment+camera+lighting 을 한 프롬프트에 다 |
| 컷 간 얼굴 드리프트 | 레퍼런스 바인딩(4각도/3~8초), 동일 디스크립터 반복 → `kling-consistency` |
| 5초+ 정체성 변화 | 첫·끝 프레임, 짧게 나눠 동일 mode 로 이어붙이기 |
| 카메라 무브 중 배경 왜곡 | `fixed lens`,`tripod`,`35mm` + 배경 명시 |
| 여분 팔다리 | 네거티브 `morphing, warping, extra limbs`, 모션 강도↓ |
| 무게 없는 뻣뻣함 | 동사 강화(`sprinting`) + 환경 상호작용(`splashing through puddles`) + 강도 7~10 |
| 뻣뻣한 머리/옷 | `hair streaming behind`, `fabric rippling`(국소 모션 명시) |
| 불안정 카메라 | 샷당 주 무브 하나 |
| 립싱크 어긋남 | 네이티브 오디오로 한 번에 생성(보이스 태그) → `kling-audio-voice` |
| 환영 객체(없던 의자 등) | 장면에 있는 것 명시 + 불필요 객체 네거티브 |
| 깜빡임/지터 | 네거티브 `flickering, distorted shadows`, 강한 광변화·바쁜 배경 회피 |
| 하이라이트 날림 | `protect highlights`, `visible shadow detail` |
| 손/손가락 왜곡 | 손동작 명시, 레퍼런스에서 손이 보이게 |

## 품질 최적화 (생성 전)

**입력 이미지**
- 최소 1024×1024, 가능하면 2K+. 저해상도를 억지로 업스케일하지 말 것(새로 만드는 게 낫다).
- 또렷한 텍스트/라벨이 필요하면 입력에서부터 고대비·선명하게.
- 잡다한 배경 제거, 노이즈 정리.

**설정**
- 텍스트·도표·라벨 가독성이 중요하면 고화질 mode(pro/4K).
- 모션 강도 **4~5**(보통)면 동적이면서 지터 최소.
- 깜빡임 지속 시: 더 높은 fps 로 뽑고 후처리에서 다운샘플.

**색 흐림(color shift) 방지**
- 입력 이미지 세트의 화이트밸런스를 맞춘다.
- 공격적 필터 회피. 후처리 LUT 는 프로젝트 전체 하나로.

**일관성 토글**
- "Bind Subject" + 레퍼런스 1~4장 → 복잡한 카메라 무브에도 외형·의상 고정(15초까지).

> 결과가 "되는지"가 아니라 "잘 나오는지"를 본다. 호출 형식이 맞아도 결과가 기대를 배신하면 실패다 — 약점이 보이면 곧장 만들지/쪼갤지/한 줄 알리고 갈지에 반영한다.
