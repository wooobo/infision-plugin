# Kling 능력·모델 선택

Kling VIDEO 3.0 / 3.0 Omni 로 무엇이 되고, 무엇을 고르고, 결과가 나쁠 때 어떻게 고치나. 방법이 못 내는 결과를 약속하지 말 것 — 길이·동작·정합을 능력에 맞춘다.

## 능력 (제약)

| 항목 | 값 |
| --- | --- |
| 영상 길이 | 3 ~ 15초 (가변) |
| 멀티샷 최대 컷 | 6 (총 길이 3~15초 내) |
| 화면비 | 16:9 / 9:16 / 1:1 |
| FPS | 최대 60 |
| 화질 | 720p(std) / 1080p(pro) / 4K(native) |
| 캐릭터용 영상 레퍼런스 | 3~8초 |
| 보이스용 오디오 레퍼런스 | 5~30초 |
| 요소 바인딩/작업 | 최대 3 |
| 프롬프트 한도 | 3.0=500자 / Omni=2,000자 |

- **네이티브 오디오·립싱크**: 영상과 동시 1-pass(후처리 아님). 대사·환경음·BGM. 5개국어(중·영·일·한·스). 멀티 화자는 `<<<voice_1>>>` 태그.
- **멀티샷/Director**: ≤6컷. Auto(단일 프롬프트→자동 컷 설계) 또는 Custom(컷별 내용·길이 지정, 컷 길이 합=총 길이).
- **정합**: VIDEO 3.0 은 3+ 인물 가능, Omni 는 1~2 요소 정합 최적.
- **native 4K**: 픽셀 단위 생성(업스케일 아님).

## 모델·옵션 선택

- **길이는 동작과 한 묶음.** 미세한 움직임=짧게(3~5초), 크고 다단계 동작=길게(10~15초). 짧은 클립에 안 담길 동작을 쓰지 말 것.
- **화질(infision 카탈로그 mode)**: 가성비 720p(std) / 기본 고품질 1080p(pro) / 최고 4K. 오디오 on 은 비용을 ~1.5배(크레딧 기준) 올린다.
- **다인물(3+) 한 장면**이면 정합이 어렵다 — 분해(인물별 생성 후 합성)나 멀티샷을 고려.
- **정체성 보존 + 외형 교체 + 움직임을 한 영상에 동시에** 요구하면 한 방에 잘 안 나온다 → 단계로 쪼개는 게 정합에 유리하다(앞 단계 결과를 다음 단계의 레퍼런스로 쓰는 흐름). 단계를 캔버스에서 어떻게 잇는지는 사용자가 정하고, 각 단계의 프롬프트만 작성한다.
- 시나리오: t2v / i2v(첫·끝 프레임) / r2v(참고 이미지 ≤7).

결과가 기대를 벗어날 때의 증상→처방, 품질 최적화는 → `kling-troubleshooting`. 입문(t2v/i2v) 기본: 프롬프트 = subject+action → scene → style/camera. 잘 조명된 정면 고해상 입력이 i2v 에 유리하고, 머리카락·물·구름·잎 같은 요소가 자연스러운 모션 앵커가 된다.

## 가격 (참고 — 미확정)

크레딧/초(블로그 기준, 공식 미검증): 1080p 8 / 720p 6 / +오디오 12·9 / voice-only 2. 길이는 정수 초 반올림. 멀티샷 추가 요금 없음. **4K 단가·크레딧→USD 는 공식 미확인** — 정확한 비용을 단정하지 말 것.
