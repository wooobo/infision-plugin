# Kling 오디오·보이스·립싱크

Kling Omni 는 영상과 오디오(대사·환경음·BGM)를 **한 번에** 생성하고 입을 맞춘다(립싱크). 별도 후처리 단계가 아니다. 어휘는 영문 그대로.

## 언제

- 인물이 말하는 장면(대사·내레이션), 환경음/Foley 가 필요한 장면, 보이스가 정체성의 일부일 때.
- 오디오를 켜면 비용이 오른다(크레딧 ~1.5배) — 후처리에서 음악을 입힐 거면 오디오 off 가 낫다.

## 대사·보이스 (프롬프트)

- 화자에 보이스 태그를 붙이고, 대사는 따옴표로:
```
The man <<<voice_1>>> says, "We need to leave now."
```
- 멀티 화자: 대화 순서대로 `<<<voice_1>>>`, `<<<voice_2>>>`.
- 감정/전달 마커가 오디오 + 표정에 반영: `whispers`, `exclaims`, `in a low, flat voice`, `nervously`.
- 말 전에 물리 동작을 먼저 쓰면 자연스럽다: `slams his hand on the table, then says, "..."`.
- 지원 언어 5종(중·영·일·한·스), 영어 악센트(미/영/인도) 등.

## 보이스 바인딩(정체성+음색)

- 영상 레퍼런스 3~8초 → 외형과 보이스 톤 동시 추출.
- 이미지 1~4장 + 별도 오디오 5~30초 → 외형 + 음색 바인딩.
- 오디오 레퍼런스 요건: 깨끗한 배경, 단일 화자, 겹치는 목소리·큰 BGM 없음.

## 환경음/Foley

- 장면 묘사에 소리 단서를 넣으면 의미적으로 매칭된 오디오가 생성된다:
```
galloping hooves on cobblestone, distant thunder rolling, soft rain on glass, low ambient chatter
```

## 믹싱(후처리로 음악을 입힐 때)

- BGM/SFX 는 내레이션 대비 **20~30% 볼륨**, 더 크면 말이 묻힌다. 오디오 더킹 적용.
- 긴 스크립트를 한 번에 TTS 돌리지 말고 문단 단위로 쪼개야 끊김이 없다(쉼표·말줄임으로 호흡).
- 클립들을 합칠 땐 fps 를 통일(예: 30)해 temporal 불일치 회피.

## 주의

- 오디오/영상을 따로 만들면 립싱크가 어긋난다 — Omni 의 네이티브 오디오로 한 번에 뽑는 게 가장 안전.
- 소셜 영상은 대부분 음소거로 먼저 본다 — 시각만으로도 전달되게 설계하고 오디오는 보강으로.
